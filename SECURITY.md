# Security Policy

This repository does not yet contain a supported production release. The
public Tiny Tapeout wrapper and DOM32 candidate have the limitations listed in
[Validation Status](VALIDATION_STATUS.md) and
[Product Status](PRODUCT_STATUS.md).

## Reporting a vulnerability

Report suspected vulnerabilities through
[GitHub private vulnerability reporting](https://github.com/shivamtiwari020505/Masked_AES_Core_Tapeout_USBdongle/security/advisories/new).
Include the affected commit or release, threat model, reproduction steps, and
observed impact. Do not include real keys, customer data, foundry-confidential
material, or other information you are not authorized to share.

Do not disclose an uncoordinated security report through a public Issue. If the
private reporting form is unavailable, open a public Issue asking for a private
contact channel without describing the vulnerability.

Receipt of a report does not create a support SLA, warranty, or bounty. Any
such terms must be part of a separate written agreement.
